## 概述

基于 plan.md 中的任务列表，逐步执行实施，直到core和当前feature的 E2E 测试用例通过。

## 执行步骤

### 1. 加载上下文

1. 读取当前分支对应的 `specs/{branch-name}/spec.md` 和 `specs/{branch-name}/plan.md`
2. 读取项目的 `CLAUDE.md` 了解项目架构和约定
3. 解析 plan.md 中的任务列表

### 2. 任务执行

按照 plan.md 中定义的阶段和顺序执行任务：

#### 执行规则

1. **按阶段执行**: 完成当前阶段所有任务后再进入下一阶段
2. **遵循依赖**: 按照任务依赖关系顺序执行
3. **实时更新**: 完成一个任务后立即在 plan.md 中标记为 `[x]`
4. **错误处理**: 遇到问题时停止并报告，等待用户指示

#### 任务完成标准

每个任务完成需满足：
- 代码已编写并保存
- 无语法错误
- 符合项目代码风格（参考 CLAUDE.md）

### 3. E2E 测试执行

#### 3.1 测试方法

请参考README.MD开始 e2e 测试，如果没有相关 e2e测试方案，请和用户确认并创建

#### 3.3 测试结果处理

**如果测试通过**:
- 标记所有测试任务为完成
- 报告成功

**如果测试失败**:
1. 分析失败原因
2. 定位问题代码
3. 修复问题
4. 重新运行测试
5. 重复直到通过

### 4. 进度报告

每完成一个任务阶段，报告：
- 已完成任务数 / 总任务数
- 当前阶段状态
- 遇到的问题（如有）

## 实施原则

### 代码质量

- 遵循 readme.md 中定义的代码风格

### 错误处理

- 遇到不明确的问题时询问用户
- 不要假设或跳过问题
- 保留错误日志便于调试

### 测试驱动

- E2E 测试是最终验收标准
- 测试失败时优先分析测试代码是否正确
- 确保测试覆盖 spec.md 中定义的 Golden Case
- **必须实现所有测试用例**: 不要因为成本或时间原因跳过 spec 中设计的测试用例，这些用例是检验功能是否正常的关键标准

## 完成标准

实施完成需满足以下所有条件：

1. **所有任务完成**: plan.md 中所有任务标记为 `[x]`
2. **E2E 测试通过**: spec.md 中定义的所有测试用例通过

> **⚠️ 重要**: 任务完成后请返回检查 plan.md 中的所有 task 是否全部完成（标记为 `[x]`）。如果有未完成的任务，必须继续执行直到全部完成！

## 输出

完成后报告：
1. 任务完成情况统计
2. E2E 测试结果
3. 修改的文件列表
