# OhMyAICodingToolbox - AI 记忆文件

本文件是 AI 编码工具的项目记忆，包含项目约定和自我学习的知识。

## 项目约定

- 所有 command 使用 `oh.` 前缀
- 使用 `CLAUDE.md` 作为项目记忆文件（Cursor 和 Claude Code 都会自动加载）

## Command 设计原则

### 核心理念

Command 应该定义**流程和原则**，而不是预设**具体技术选型**。每个项目都有自己的架构和技术栈，command 需要适应项目而不是强制项目适应 command。

### 应该包含的内容

| 类型 | 示例 |
|------|------|
| 执行流程 | 加载上下文 → 分析现有架构 → 制定方案 → 执行任务 |
| 必做事项 | 探索项目代码、回归测试、记忆更新 |
| 通用原则 | 复用优先、保持一致、最小新增 |
| 输出规范 | 需要生成什么文件、报告什么内容 |

### 不应该包含的内容

| 类型 | 示例 |
|------|------|
| 具体框架选型 | Playwright vs Cypress、pytest vs Jest |
| 预设代码模板 | Page Object 类的具体实现代码 |
| 固定目录结构 | tests/e2e/pages/、tests/e2e/specs/ |
| 具体命令示例 | npx playwright test、npm run test:e2e |

### 设计检查清单

创建或修改 command 时，检查以下问题：

- [ ] 是否要求先探索项目现有架构？
- [ ] 是否强调复用现有资源？
- [ ] 是否避免了预设具体技术选型？
- [ ] 是否让 AI 根据项目实际情况做决策？
- [ ] 是否包含了必做的验证步骤（测试、回归）？
- [ ] 是否包含了记忆更新机制？

## 可用 Commands

### 应用开发 (app/)

| Command | 用途 |
|---------|------|
| `oh.specify.app` | 创建功能规范（spec.app.md） |
| `oh.plan.app` | 创建技术方案（plan.app.md） |
| `oh.implement.app` | 执行方案直到测试通过 |

### E2E 测试项目 (e2e/)

| Command | 用途 |
|---------|------|
| `oh.specify.e2e` | 创建测试规范（spec.e2e.md） |
| `oh.plan.e2e` | 创建测试方案（plan.e2e.md） |
| `oh.implement.e2e` | 执行测试实现直到通过 |

## 自我学习机制

每个开发周期结束后，需要智能更新本记忆文件。

### 记忆更新触发条件

| 条件 | 优先级 |
|------|--------|
| 同一错误出现 2 次以上 | 高 |
| 解决方案需要大量调试时间 | 高 |
| 现有记忆内容不再准确 | 高 |
| 发现意外行为或边界情况 | 中 |
| 新的架构模式或技术选型 | 中 |
| 发现更好的实践方法 | 中 |

### 应该记录的内容

- 花费大量调试时间的错误及解决方案
- 与文档描述不符的行为
- 应该复用的模式
- 项目特定的知识

### 不应该记录的内容

- 通用编程知识
- 不太可能再次发生的一次性修复
- 临时解决方案
